本システムは、ローカルのフォルダ内にある各種ファイル（Office文書、動画・音声）をRAG（検索拡張生成）の基盤データとして利用しやすい形式に自動変換し、Amazon S3へ安全かつ効率的に同期アップロードするためのツール一式です。

`run_process.bat` を実行するだけで、ライブラリのセットアップからファイルの処理、S3への同期までをワンストップで行うことができます。

## 🛠 システムの処理フロー

1. 一括セットアップ: 実行に必要なライブラリが不足している場合、自動で一括インストールします。
2. ファイルの自動変換 (`clean.py`):
Office文書: PDF形式に自動変換（LibreOffice headlessを使用）。
動画・音声ファイル: 音声データを抽出し、AI（OpenAI Whisper）によってテキスト化（.txt）。
出力: 対象フォルダの直下に、フォルダ名と同名の「変換成果物フォルダ」を自動生成して格納します。


3. S3への同期アップロード (`milnewupload.py`):
生成された成果物フォルダを、指定のAmazon S3バケットへアップロードします。
差分検知: サイズと更新日時を比較し、変更のないファイルはアップロードをスキップします。
スマート同期削除: ローカルで削除された古いファイルをS3上からも自動削除します。

## 📂 フォルダ構成（イメージ）

スクリプトを配置したディレクトリでバッチを実行した際の、ローカルの処理イメージです。

```text
📁 任意の作業フォルダ（例：ナレッジ変換）
 │  📄 提案資料.pptx
 │  📄 議事録.xlsx
 │  📄 セミナー動画.mp4
 │
 └─📁 ナレッジ変換 （★自動生成される成果物フォルダ）
     │  📄 提案資料.pdf         (←自動変換)
     │  📄 議事録.pdf           (←自動変換)
     │  📄 セミナー動画.txt     (←自動文字起こし)

```

## ✨ 主な機能

重複処理のスキップ: すでに変換後のPDFやTXTが存在する場合、処理をスキップして高速化します。
差分アップロード＆同期削除: AWSへの転送量を最小限に抑えつつ、ローカルとS3の状態をクリーンに保ちます。
一時ファイルの自動クリーンアップ: 動画からの音声抽出時に作成される一時ファイル（`_temp.mp3`）は処理後に自動削除されます。

## 📋 前提条件・必要な環境

本システムを利用するには、事前に以下の環境がローカルPCにセットアップされている必要があります。

1. Python 3系がインストールされ、環境変数 `PATH` に追加されていること。
2. LibreOffice
事前に公式サイトからダウンロードしてインストールしてください。
OfficeファイルをPDFに変換するために使用します。インストール先が異なる場合は、`clean.py` 内の `LIBREOFFICE_PATH` を修正してください。

3. AWS S3の転送先設定
`milnewupload.py` のデータ転送先を個別に設定する必要があります。

4. AWS 認証情報の設定
`boto3` が動作するよう、環境変数（`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`）や、AWS CLIでの認証が完了している必要があります。

5. FFmpeg
動画・音声処理に必要です。`clean.py` 内で一時的にパスを通す処理が含まれています。

## 🚀 使い方

### 1. スクリプトの配置

同一のフォルダ内に以下の3つのファイルを配置します。

`run_process.bat`
`clean.py`
`milnewupload.py`

### 2. システムの実行

1. `run_process.bat` をダブルクリックして起動します。
2. 画面の指示に従い、処理したいローカルのフォルダパスを入力（またはドラッグ＆ドロップ）して `Enter` を押します。  
※パスを何も入力せずに `Enter` を押した場合、デフォルトでデスクトップの「ナレッジ変換」フォルダが対象になります。

3. 必要なライブラリの確認、ファイルの変換処理、S3へのアップロードが自動的に順次実行されます。
4. 「すべての処理が完了しました！」と表示されたら、何かキーを押してウィンドウを閉じます。

## ⚙️ 設定のカスタマイズ（開発者向け）

環境や要件に合わせて設定を変更する場合は、各スクリプトの以下の箇所を修正してください。

### 1. S3の保存先変更 (`upload.py`)

上部の変数で保存先のバケット名とフォルダ（プレフィックス）を指定しています。

```python
# ==========================================
# 保存先URLを設定
# ==========================================
bucket_name = 'rag-document-535179725289-ap-northeast-1-an'
s3_folder = 'TEST/ｔｔｔ/' 

```

### 2. FFmpegのパス変更 (`clean.py`)

環境依存のFFmpegバイナリのパスが指定されています。必要に応じてお使いの環境のパスに書き換えてください。

```python
os.environ["PATH"] += os.pathsep + r"C:\Users\s_ise\Desktop\調査用\knowledge-base-rag\ffmpeg-2026-06-15-git-44d082edc8-essentials_build\bin"

```

### 3. Whisperモデルの変更 (`clean.py`)

現在は軽量な `"base"` モデルが指定されています。文字起こしの精度を上げたい場合は `"small"`, `"medium"`, `"large"` 等に変更可能です（※処理時間とマシン負荷が増加します）。

```python
model = whisper.load_model("base")

```

## ⚠️ 注意事項

S3上のファイル同期削除について:
`milnewupload.py` の仕様により、指定したS3上のフォルダ配下において、ローカルに存在しないファイル（過去に処理して今回は消したファイルなど）はS3側からも削除されます。誤消去を防ぐため、S3上の同じ場所に手動で別ファイルをアップロードしないよう注意してください。
一時的なロックファイルの除外:
Microsoft Officeが生成する一時ファイル（`~$` から始まるファイルなど）は、変換エラーを防ぐため自動的にスキップされる仕様になっています。
