## 1. 事前準備

### Pythonライブラリのインストール

同期を高速に行い、進捗状況を可視化するために `tqdm` のインストールを推奨します。

```bash
pip install tqdm boto3

```

### AWS認証情報の設定

このスクリプトは `boto3` を使用しています。PC上に AWS の認証情報が正しく設定されていることを確認してください。

`aws configure` コマンドで `AWS Access Key ID` と `Secret Access Key` が設定済みであること。
または、環境変数やIAMロールで権限が適切に付与されていること。

## 2. 実行方法

コマンドプロンプトやPowerShellで、以下の形式で実行します。

```bash
python milaupload.py "同期したいフォルダのパス"

```

### 実行例

```bash
# Googleドライブなどのパスを指定する場合
python milaupload.py "G:\マイドライブ\案件フォルダ"

```

**手動実行時:*コンソール上に進捗バー（`[██████░░░░] 60%`）が表示されます。
**自動実行時（ログ出力）:*進捗バーは表示されず、処理内容が標準出力にログとして書き出されます。

## 3. タスクスケジューラでの自動運用

Windowsの「タスクスケジューラ」で定期実行させるための設定手順です。

1. **プログラム/スクリプト**: `python`
2. **引数の追加**:
```text
C:\パス\to\milaupload.py "G:\マイドライブ\案件フォルダ" >> C:\パス\to\sync.log 2>&1

```


3. **開始（オプション）**: `C:\パス\to\` （スクリプトが置いてあるフォルダ）

※ `>> sync.log 2>&1` を末尾に付けることで、処理結果が `sync.log` ファイルに自動保存され、後から成否を確認できます。

## 4. 仕様の注意点

**ミラーリング機能**: ローカル側に存在しないファイルは、S3側からも削除されます。**重要なファイルを誤って消さないようご注意ください。**
**Googleドライブ連携**:
Googleドライブの「デスクトップ版」を使用している場合、対象フォルダを右クリックして「オフラインで使用する」設定にしてください。
「ストリーミング」設定のままオンライン専用ファイルとして扱うと、スクリプトがファイルを認識できず、削除同期が誤作動する可能性があります。

**安全性**: S3上の情報取得に失敗した場合は、安全のため即座に処理を中断する設計になっています。
